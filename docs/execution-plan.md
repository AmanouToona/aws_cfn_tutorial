# 実行計画（初心者向け）

## 1. 目標

このプロジェクトは、AWS の学習を目的としたサーバーレス SaaS 風アプリの構築です。
初心者でも進めやすいように、まずは「動く最小構成」を作り、その後にセキュリティ機能を追加していく方針で進めます。

## 2. 実行方針

- まずは 1 つずつ確認しながら進める
- 各ステップは AWS CLI で確認する（原則、AWS コンソールを使わない）
- 1 つの変更で複数の要素を一気に増やさない
- 失敗しても復旧しやすいように、スタックを小さく分ける

追加方針:

- デプロイは CloudFormation / SAM + AWS CLI のみで実行する
- 手動クリック作業をなくすため、作成・更新・削除をすべてテンプレート化する

## 3. 実施順序

### Phase 0: 準備

1. AWS アカウントを用意する
2. IAM ユーザーまたはロールを作成する
3. AWS CLI をインストール・設定する
4. SAM CLI / Node.js / Python を導入する
5. GitHub アカウントを作成する

確認ポイント:

- AWS CLI で `aws sts get-caller-identity` が成功する
- 以降の作業で使うリージョンを決める（例: ap-northeast-1）

### Phase 1: Bootstrap スタック

1. デプロイ用の S3 バケットを作成する
2. フロントエンド用の S3 バケットを作成する
3. CloudFormation 実行用の IAM ロールを作成する
4. これらを CloudFormation で管理する

目的:

- 後続のデプロイに必要な土台を作る

### Phase 2: 最小構成のバックエンドを作る

1. API Gateway を作る
2. Lambda 関数を作る
3. DynamoDB テーブルを作る
4. 1 つの公開 API を動かす

最初の目標:

- `GET /hello` などの簡単な API が動くこと

### Phase 3: フロントエンドを作る

1. 静的サイト用の S3 バケットを準備する
2. CloudFront を設定する
3. S3 から配信できるようにする
4. 画面を表示できることを確認する

最初の目標:

- ブラウザでページが表示されること

### Phase 4: WAF を追加する

1. Web ACL を作る
2. CloudFront もしくは API Gateway に紐づける
3. 不正トラフィックがブロックされることを確認する

目的:

- 公開エンドポイントに防御層を追加する

### Phase 5: Cognito 認証を追加する

1. ユーザープールを作る
2. フロントエンドからログインできるようにする
3. 認証済み API を確認する
4. PKCE を使う構成にする

目的:

- ログイン・ログアウト・認証付き API の流れを理解する

### Phase 6: レート制限を追加する

1. 未認証ユーザー用の制限を作る
2. 認証済みユーザー用の制限を作る
3. 429 エラーが返ることを確認する

目的:

- API の利用制御を体験する

### Phase 7: CI/CD を整える

1. GitHub Actions を作る
2. OIDC で AWS 認証する
3. コード変更時に自動デプロイされるようにする
4. CloudFront のキャッシュ無効化も自動化する

目的:

- 手作業を減らし、再現性を高める

### Phase 8: 監視と運用

1. CloudWatch Logs を確認できるようにする
2. WAF / CloudFront のログを有効化する
3. 主要なエラーを追跡できる状態にする

### Phase 9: 完全削除（チュートリアル終了処理）

1. CloudFront ディストリビューションの無効化と削除を実行する
2. frontend-dispatch スタックを削除する
3. application スタックを削除する
4. waf スタックを削除する
5. bootstrap スタックを削除する
6. 残存 S3 オブジェクトと CloudWatch ロググループを確認し、残っていれば削除する
7. `aws cloudformation list-stacks` で関連スタックが `DELETE_COMPLETE` になっていることを確認する

目的:

- 課金リスクを残さずに、学習環境を安全に閉じる

## 4. 1 つずつ進めるためのおすすめ順番

初心者には、以下の順番が特におすすめです。

1. AWS の基本設定
2. 最小の API を作る
3. その API をブラウザから確認する
4. S3 + CloudFront で静的サイトを配信する
5. WAF を追加する
6. Cognito でログインする
7. レート制限を追加する
8. GitHub Actions で自動化する

## 5. 各フェーズでの成功条件

- 1 つのステップが完了したら、必ず確認する
- 確認できたら次に進む
- できない場合は、その段階で止まって原因を調べる

## 6. 初心者向けの注意点

- まずは「完全版」ではなく「動く最小構成」を目指す
- CloudFormation のテンプレートは小さく保つ
- 1 回で全部作ろうとしない
- AWS の費用がかかるので、不要なリソースはすぐ削除する

## 7. 最初の 1 週間のおすすめ作業

- AWS アカウントと CLI の設定
- 最小の Lambda + API Gateway を作る
- 画面表示できる静的サイトを作る
- その後に WAF と Cognito を追加する

## 8. 次にやること

この計画に沿って、まずは「Phase 0: 準備」と「Phase 1: Bootstrap スタック」から始めます。

補足:

- 具体的な CLI コマンドは `deploy-and-destroy-process.md` にまとめる
